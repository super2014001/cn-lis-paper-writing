# 回复审稿意见指南

## 目标
回复审稿意见的核心任务，不是“证明自己没错”，而是以礼貌、具体、可核对的方式说明：
1. 你是否理解了审稿人的问题
2. 你做了什么修改
3. 修改具体体现在哪里
4. 哪些意见你采纳了，哪些没有完全采纳，以及原因是什么

## 基本原则
1. 语气要礼貌、克制、专业。
2. 每条意见最好逐条回应，不要笼统合并。
3. 先感谢，再回应，再说明修改位置。
4. 如果没有采纳某条建议，应说明学理原因、数据限制或研究边界，避免情绪化辩解。
5. 回复必须与正文修改对应，不能只写“已修改”而不说明如何修改。

## 通用结构
### 模板 A：感谢—理解—修改—定位
- 感谢审稿人提出意见
- 重述你理解的问题
- 说明你做出的修改
- 标出修改位置

推荐句式：
- 感谢审稿人提出这一重要意见。
- 根据该意见，本文已在……部分作出修改。
- 具体而言，补充了……，并对……进行了重写。
- 相关修改见第 X 页第 X 段 / 第 X 节。

### 模板 B：部分采纳型回应
适用于无法完全按建议修改的情况。

推荐句式：
- 感谢审稿人的建议。我们充分认同该意见所强调的……
- 受限于……，本文暂无法……
- 因此，本文未对……作全面展开，但已在……部分补充说明，并在结论中将其列为后续研究方向。

## 审稿意见的常见类型与回应策略
### 一、认为研究问题不清
回应重点：
- 是否重写了引言结尾
- 是否明确补出了研究问题 / 研究目标
- 是否调整了题目、摘要与引言的一致性

### 二、认为文献综述不足
回应重点：
- 是否补充了关键文献
- 是否重组了综述结构
- 是否更明确地提出研究缺口

### 三、认为方法说明不充分
回应重点：
- 是否补写样本来源、筛选规则、分析步骤、参数或编码依据
- 是否增加了方法合理性说明

### 四、认为结果讨论不够深入
回应重点：
- 是否补充了原因解释
- 是否加入了与既有研究的比较
- 是否强化了实践启示

### 五、认为结论与启示过强
回应重点：
- 是否弱化了超出证据边界的表述
- 是否补充了局限与适用边界

## 常见错误
1. 只写“感谢意见，已修改”，没有具体内容。
2. 语气防御性太强，如“我们认为审稿人误解了……”。
3. 只在回复信里解释，却没有在正文中同步修改。
4. 不标注修改位置，增加编辑和审稿人核对成本。
5. 把不同意见混在一起回复，造成结构混乱。

## 默认输出要求
当处理审稿意见回复时，应输出：
1. 逐条回应草稿
2. 每条回应的语气优化版本
3. 对应的正文修改建议
4. 可直接发送的整封回复文本

## 好回复的判断标准
一份较好的回复通常满足：
- 礼貌克制
- 逐条对应
- 具体到修改动作和位置
- 既尊重意见，又守住研究边界
