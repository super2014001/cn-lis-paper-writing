# 结论写作指南

## 目标
结论部分的任务不是简单重复摘要，也不是把所有优点再说一遍，而是有层次地收束全文，明确回答：
1. 本文最重要的发现是什么
2. 这些发现对学术理解有什么补充
3. 对图书馆实践、信息服务或公共文化治理有什么启示
4. 本文还有哪些边界与局限
5. 后续研究还可以往哪里推进

## 基本原则
1. 结论应建立在前文结果与讨论基础上，不能脱离证据另起炉灶。
2. 结论要有层次，至少区分研究发现、理论意义、实践启示、局限与展望。
3. 实践启示应尽量具体到机构、对象、流程、服务场景或制度安排。
4. 不要把“做了什么”直接等同于“创新贡献”。
5. 不要在结论中引入正文未讨论的新材料、新观点或新判断。

## 常用结构
### 模板 A：发现—意义—启示—局限—展望
适用于大多数中文图书情报论文。

- 发现：概括 2—4 条核心发现
- 意义：说明这些发现对现有研究补充了什么
- 启示：说明对实践改进有什么帮助
- 局限：说明研究边界
- 展望：说明后续可深化的方向

### 模板 B：按研究问题回收
适用于研究问题清晰的论文。

- 逐一回应研究问题
- 概括总体认识
- 提出对应实践建议
- 补充局限与展望

## 结论中应重点处理的内容
### 一、研究发现
推荐写法：
- 本文研究表明，……
- 综合分析可见，……
- 研究结果主要体现为以下几个方面：……

要求：
- 只保留最重要的发现
- 不重复方法过程和枝节结果

### 二、理论意义
可从以下角度展开：
- 补充了什么研究对象或情境
- 解释了什么机制或关系
- 改进了什么分析框架或研究视角

必须避免：
- 只写“丰富了相关研究”而不具体说明

### 三、实践启示
可从以下方向落地：
- 图书馆服务设计与优化
- 用户支持与信息干预
- 资源组织与资源建设
- 平台治理与流程改进
- 公共文化服务与制度完善

推荐句式：
- 从实践层面看，相关机构可从……方面优化……
- 对于……场景，研究提示应重点关注……
- 结合本文发现，可进一步完善……机制。

### 四、局限与展望
常见局限包括：
- 样本范围有限
- 数据时段有限
- 研究方法较单一
- 案例情境具有特殊性
- 对机制解释仍不够充分

展望写法：
- 后续研究可进一步扩大样本范围 / 引入多源数据 / 比较不同场景。
- 未来可围绕……开展更细致的机制分析或跟踪研究。

## 常见问题
### 问题 1：结论只是摘要复述
处理：减少背景和方法信息，改为概括发现与意义。

### 问题 2：启示空泛
处理：把启示落实到“谁在什么场景中改进什么”。

### 问题 3：没有局限
处理：至少说明 1—2 条真实边界，增强可信度。

### 问题 4：过度拔高
处理：凡是正文没有充分证据支撑的强结论，必须降级表述。

## 结论改写时的默认输出要求
当处理结论时，应输出：
1. 当前结论存在的问题
2. 推荐的层次结构
3. 改写后的结论
4. 哪些地方仍需补证据或弱化表述

## 好结论的判断标准
一篇较好的结论通常满足：
- 能简洁收束全文
- 能区分发现、意义、启示、局限
- 启示具体可落实
- 不超过前文证据边界
