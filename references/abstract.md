# 摘要写作指南

## 目标
中文图书情报论文摘要的核心任务不是“写得宏大”，而是用尽可能短的篇幅说明：
1. 研究什么问题
2. 为什么值得研究
3. 用什么材料或方法研究
4. 发现了什么
5. 这些发现意味着什么，尤其对图书馆业务改进、信息服务优化、资源建设、用户支持或政策制定有什么启示

## 基本原则
1. 摘要必须能独立传达论文主要内容。
2. 摘要不是引言压缩版，也不是结论改写版。
3. 摘要不能只写背景和意义，不写方法和结果。
4. 摘要中的“创新”“启示”“价值”必须建立在具体发现基础上。
5. 摘要应避免作者名单式文献回顾与空泛政策口号。
6. 摘要中的实践启示应尽量具体说明指向何种管理改进、服务优化、资源建设或制度完善，而不是笼统写成“具有实践意义”。

## 通用结构
大多数中文图书情报论文摘要可采用以下五步：

### 模板 A：背景—问题—方法—发现—启示
适用于绝大多数论文。

- 背景：说明研究场景或现实问题
- 问题：点出本文具体研究对象与核心问题
- 方法：说明数据来源、样本、方法或分析框架
- 发现：概括 2—4 条关键结果
- 启示：说明理论价值，并优先写清实践启示落点

### 模板 B：对象—数据—方法—结果—结论
适用于更偏实证、计量、调查的论文。

- 对象：研究的对象或范围
- 数据：数据来源、样本量、时间范围
- 方法：使用的分析方法
- 结果：主要发现
- 结论：简洁总结意义与实践指向

### 模板 C：问题—综述不足—研究设计—主要发现
适用于问题导向、综述导向较强的论文。

- 问题提出
- 指出现有研究不足
- 说明本文如何展开研究
- 给出主要结论与实践启示

## 分类型模板

### 一、文献计量 / 知识图谱类摘要
建议结构：
- 指出研究主题或领域背景
- 说明数据来源、时间范围、样本数量
- 说明使用的计量或可视化方法
- 概括研究热点、演化路径、前沿趋势
- 给出学科启示或实践启示

可用句式：
- 以某数据库收录的某类文献为样本，运用……方法，对……进行分析。
- 研究发现：……
- 结果表明，该领域呈现出……特征，其演化趋势主要表现为……
- 研究结果可为……领域的资源配置、服务设计或研究布局提供参考。

必须避免：
- 只说“进行了可视化分析”，不说样本与范围
- 只说“总结热点”，不写具体发现
- 把摘要写成软件操作说明

### 二、问卷 / 访谈 / 用户行为研究类摘要
建议结构：
- 交代研究对象与研究问题
- 说明样本、调查工具或访谈方法
- 说明分析方法
- 提炼关键影响因素、机制或关系
- 概括管理建议或服务启示，并尽量点明适用对象与应用场景

可用句式：
- 以……为研究对象，通过问卷调查 / 访谈 / 实验收集数据。
- 运用……模型 / 方法分析……
- 研究发现，……对……具有显著影响。
- 基于上述发现，可从……方面优化相关服务与支持机制。

必须避免：
- 不交代样本或对象
- 结果只有“有影响”，没有方向、机制或层次
- 在摘要里加入过多统计细节

### 三、政策 / 案例 / 内容分析类摘要
建议结构：
- 点出研究背景与研究对象
- 说明材料来源与分析框架
- 概括分析得到的结构、类型、特征或问题
- 提出结论与建议，尤其强调对图书馆实践、信息服务或公共文化治理的启示

可用句式：
- 基于……文本 / 案例 / 政策材料，构建……分析框架。
- 研究发现，当前……主要表现为……
- 在此基础上，提出……
- 相关结论可为……制度完善、服务设计或平台治理提供依据。

必须避免：
- 只讲政策背景，不讲分析对象
- 只给建议，不交代结论从何而来
- 使用空泛措辞替代分析结果

## 摘要常见问题

### 问题 1：只有意义，没有研究内容
典型表现：
- 开头大量写时代背景
- 中间没有方法
- 结尾直接说“具有重要意义”

处理方式：
- 删除泛背景
- 补入研究对象、方法与核心结果

### 问题 2：只有方法，没有发现
典型表现：
- 连续写“采用……方法”“构建……模型”
- 看不出论文最终发现了什么

处理方式：
- 至少补 2—3 条关键发现

### 问题 3：摘要像目录
典型表现：
- “首先分析……其次研究……最后提出……”
- 只写做了什么，不写得出了什么

处理方式：
- 用“研究发现”“结果表明”替代程序性表述

### 问题 4：启示空泛
典型表现：
- “具有重要实践意义”
- “可为相关工作提供参考”

处理方式：
- 指明为谁提供参考
- 说明可改进什么流程、服务、机制或资源建设环节

### 问题 5：摘要与正文不一致
典型表现：
- 摘要中宣称“构建模型”“提出机制”，正文实际上没有完整展开
- 摘要中结论过强，正文证据不足

处理方式：
- 检查摘要中的每个强结论是否都能在正文中找到依据

## 摘要改写任务的默认输出要求
当处理摘要时，应输出：
1. 当前摘要存在的问题
2. 推荐摘要结构
3. 改写后的摘要
4. 是否还缺少样本、方法、结论等关键信息

## 摘要压缩规则
当用户要求“压缩摘要”时：
1. 优先保留研究对象、方法、主要发现
2. 压缩大而空的背景句
3. 合并重复意义句
4. 不删除能体现研究边界的关键信息

## 好摘要的判断标准
一篇较好的摘要通常满足：
- 不看正文也能知道研究内容
- 能看出研究对象、方法与结论
- 没有明显空话套话
- 语言简洁，但信息完整
- 能看出实践启示具体落向何处
