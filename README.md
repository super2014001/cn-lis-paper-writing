# cn-lis-paper-writing

> **一句话定位**：专门为中文图书情报领域学术论文打造的 AI Skill 包——帮你诊断问题、重构结构、改写文本、回复审稿，覆盖投稿全流程。

---

## 这个 Skill 能帮你做什么

| 你的任务 | Skill 的动作 |
|---|---|
| 摘要太长/太空洞 | 按"目的—方法—结果—结论"框架压缩改写 |
| 引言不知怎么展开 | 诊断背景—问题—缺口—贡献的逻辑链 |
| 文献综述像堆砌 | 重组为"议题演进—研究缺口—本文定位"结构 |
| 方法交代不清 | 补充对象、材料、步骤、工具、判断依据 |
| 结果和结论脱节 | 检查研究问题—方法—结果—结论的闭环 |
| 实践启示太抽象 | 落实到服务改进、管理优化、政策完善、资源建设层面 |
| 收到审稿意见不知如何回 | 生成礼貌、克制、逐条有据的回复草稿 |
| 投稿前不放心 | 执行结构性与规范性全文自检清单 |

**本 Skill 不做的事**：不写学位论文、不做基金申报书、不处理纯技术算法论文、不生成宣传稿或工作总结。

## 快速上手

典型对话入口
用户：我的摘要写完了，帮我看看有没有问题
Skill：[诊断问题] → [给出改写提纲] → [输出改写文本] → [说明修改理由]

用户：审稿人说我文献综述堆砌，怎么办
Skill：[识别堆砌类型] → [重组议题主线] → [补研究缺口] → [生成回复草稿]


**建议使用流程**

1. 把你当前处理的章节粘进来，说明所投期刊
2. Skill 会先诊断主要问题（不超过 5 条）
3. 给出重写提纲，确认后再给出改写文本
4. 改写完成后自动执行 reverse outline 检验
5. 全文完成后用 `reviewer-checklist.md` 通篇自检

## 设计原则

本 Skill 以以下四条标准作为"改写质量"的判断依据：

1. **研究问题清楚**：读者一眼能看出你在解决什么问题
2. **论证链条完整**：问题—方法—结果—结论闭环，无跳跃
3. **表达符合学术规范**：符合中文图书情报期刊的表达习惯
4. **实践启示能落地**：可以指向具体的图书馆服务、资源建设或政策改进措施

不以华丽表达为目标，不鼓励"具有重要意义""填补空白"等无内容的套话。
